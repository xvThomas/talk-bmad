# Decision Log — AG-UI Protocol Server (`talk serve`)

| #   | Date       | Decision                                                                      | Rationale                                                                                                                                                                                                                                                                                                     | Alternatives Rejected                                                                                                                                        |
| --- | ---------- | ----------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| D-1 | 2026-06-20 | Use `net/http` stdlib (Go 1.22+ ServeMux) — no Fiber, no external router      | Project rules forbid external routers. Performance difference irrelevant: LLM calls take seconds, HTTP parsing takes microseconds. `net/http` ServeMux pattern matching is sufficient for the 3-4 routes needed.                                                                                              | Fiber (fast but incompatible with MCP SDK `http.Handler`, project rule violation)                                                                            |
| D-2 | 2026-06-20 | No token-level streaming in v1 — single TEXT_MESSAGE_CONTENT event            | `Complete()` is synchronous. Token streaming requires a new streaming interface through the entire stack (LLM client → ConversationManager → SSE encoder). Marginal UX benefit for conversational tool use. Deferred to Phase 6.                                                                              | Implement streaming from day 1 (high complexity, low value for tool-heavy use cases)                                                                         |
| D-3 | 2026-06-20 | One server instance = one agent configuration                                 | Eliminates multi-agent routing complexity. Each agent has its own system prompt, tools, and LLM config. Multiple agents = multiple deployments. Simple, debuggable, independently scalable.                                                                                                                   | Multi-agent router with config selection (over-engineering for initial scope)                                                                                |
| D-4 | 2026-06-20 | Read configuration from existing CLI store                                    | Agent configuration (MCP URLs, system prompt, LLM provider) is already persisted by the CLI. No reason to duplicate. Operator uses CLI to configure, then runs `talk serve`.                                                                                                                                  | Separate config file/env vars (duplication, drift risk)                                                                                                      |
| D-5 | 2026-06-20 | Authentication deferred — architecture must support middleware insertion      | CopilotKit works without auth. v1 targets local/private networks. Auth middleware can be inserted at the `http.Handler` level without changing any handler code when needed.                                                                                                                                  | Implement auth from day 1 (premature for single-developer private deployment)                                                                                |
| D-6 | 2026-06-21 | AG-UI protocol over custom WebSocket protocol                                 | AG-UI is an established open-source standard (14k+ stars). CopilotKit provides full client implementation. Using a standard avoids inventing a protocol and writing client code. SSE is sufficient for server→client streaming.                                                                               | Custom WebSocket protocol (more work, no ecosystem)                                                                                                          |
| D-7 | 2026-06-21 | `talk serve` subcommand in existing binary (not separate binary)              | Reuses all existing domain code directly. No inter-process communication needed. Consistent with monorepo pattern where each module has one entry point.                                                                                                                                                      | Separate `talk-server` binary (code duplication or complex shared library)                                                                                   |
| D-8 | 2026-06-24 | Model selection is per-request via `forwardedProps.model`, not backend config | The model is a user choice, not an agent configuration. Same principle as in CLI mode where users pick the model. Agent config (static) = system prompt + MCP servers + API keys. Model (dynamic) = chosen by frontend per request via AG-UI `forwardedProps`. CopilotKit supports `forwardedProps` natively. | Default model in backend config (contradicts user choice principle); custom HTTP header (not AG-UI standard); query parameter (mixes transport and protocol) |
