## Decisions

[2026-06-28] DECISION: Separate git repository (`talk-ui`) — keeps frontend concerns isolated, independent CI, integrates via VS Code workspace.

[2026-06-28] DECISION: pnpm as package manager — fastest, strict by default (no phantom deps), standard for Vite projects in 2026.

[2026-06-28] DECISION: ESLint flat config with `typescript-eslint/strict` — modern standard (ESLint 9+), better TS support than legacy Airbnb config.

[2026-06-28] DECISION: Dark theme only (no toggle) — reduces complexity for V1, matches target aesthetic (Claude chat style).

[2026-06-28] DECISION: Models hardcoded in frontend — avoids backend dependency for V1. Future story will add `GET /models` endpoint.

[2026-06-28] DECISION: WCAG 2.1 AA structural best effort from day one — prevents accessibility debt accumulation. No formal audit V1.

[2026-06-28] DECISION: Message-level streaming (not token-level) — matches backend implementation, simpler rendering logic.

[2026-06-28] DECISION: Tool messages collapsed by default — reduces visual noise while keeping information accessible.

[2026-06-28] DECISION: Interrupt "Continue" button inline in flow — new user message after interrupt = new question, not a resume. Clear mental model.

[2026-06-28] DECISION: TanStack Router included despite single route V1 — prepares for multi-page navigation (sessions, settings) without refactoring later.

[2026-06-28] DECISION: README updated with each story — living documentation, always reflects current state.

[2026-06-28] DECISION: CopilotKit styling approach: mixed (override defaults with targeted Tailwind CSS) — avoids ejecting from CopilotKit while maintaining design control.

[2026-06-28] DECISION: Cancel behavior = Option B — question stays in history, partial response disappears, "Retry" button on cancelled question copies it back to input. Standard UX (matches Claude/ChatGPT) + convenience shortcut.

[2026-06-28] DECISION: Layered data architecture (HTTP fetch client → React Query hook + Zod → Component) — separates concerns, allows future auth header injection at HTTP layer without touching hooks or components.

[2026-06-28] DECISION: CopilotKit integration via `agents__unsafe_dev_only` + `HttpAgent` for V1 — verified that CopilotKit is natively AG-UI. No runtime needed for local dev. Production (`selfManagedAgents`) requires Enterprise license — revisit at deploy time.

[2026-06-28] DECISION: Custom AG-UI SSE client as final epic — develop all features first using CopilotKit's built-in transport (fast iteration), then replace with a custom Fetch-based SSE client implementing `AbstractAgent` interface. Removes production licensing dependency while keeping MIT-licensed React components.
